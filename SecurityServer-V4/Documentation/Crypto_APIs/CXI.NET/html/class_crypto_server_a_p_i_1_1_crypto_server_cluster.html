Detailed Description

This class implements an interface to a cluster of Utimaco's Hardware Security Moduls "CryptoServer".

Only one CryptoServer is connected at a time but in case of a communication error the API automatically switches to the another device and tries to execute the command on that device (failover).

The API tries to execute the command on every device and returns to the caller if the command could either be executed successfully or if it wasn't able to execute the command on any device.

The API begins with the first (primary) device in the given device list and switches to other devices in the given order. Once switched to another device the API basically stays at this device. Optionally a fallback interval can be given that causes the API to try to reconnect to the primary device after the interval time has expired (see fallbackInterval).

Note
The cluster API stores the authentication credential (password) of the user in order to automatically restore a lost session. Therefore the cluster API only supports password authentication, but no signature authentication.

In case of communication errors the API - silently - switches to another device and does not report the error to the calling application. If the calling application wants to get noticed about errors or state changes it can implement an event handler interface, see CryptoServerCluster::StateChanged, CryptoServerCluster::ErrorOccurred

Note
Use the class CryptoServer instead of CryptoServerCluster if a dedicated CryptoServer should be adressed!
Example (C#):

The following code illustrates the usage of the class CryptoServerCluster:

using System;
using CryptoServerAPI;
namespace CryptoServerDemo
{
class Program
{
// event handler interface
static void StateChanged(String device, CryptoServerCluster.DeviceState state)
{
Console.WriteLine("I: state of " + device + " changed to: " + state.ToString());
}
static void ErrorOccurred(String device, int err, String where, String message)
{
Console.WriteLine("E: error occurred on " + device + " at " + where
+ "\ncode: 0x" + String.Format("{0,8:X}", err)
+ "\nmessage: " + message);
}
static void Main(string[] args)
{
// register event handler
CryptoServerCluster.SetEventHandler(StateChanged, ErrorOccurred);
// create configuration object
CryptoServerConfig config = new CryptoServerConfig("./MyConfig.cfg");
config.Dump();
CryptoServerCluster csc = null;
try
{
// create new CryptoServer object with the given configuration
csc = new CryptoServerCluster(config);
csc.Logon("CXI_HMAC", "utimaco");
// get resulting authentication state
Console.WriteLine("AuthState: " + String.Format("{0:X}", csc.GetAuthState()));
// send reverse echo command (sfc 1) to CMDS firmware module (fc = 0x83)
byte[] command = System.Text.Encoding.ASCII.GetBytes("Hello World");
byte[] answ = csc.Exec(0x83, 1, command);
CryptoServerUtil.XTrace("answ", answ);
// execute other commands ...
}
catch (CryptoServerException ex)
{
Console.WriteLine(ex.Message);
}
finally
{
if (csc != null) csc.Dispose();
}
}
}
}

CryptoServerCluster ( array< String^ >^  devices,

int  read_timeout 
)

Creates a new connection to a CryptoServer cluster.

Internally only one CryptoServer is connected at a time. The API provides a failover mechanism that switches to another device in case of connection failures and returns as soon as a device has been successfully connected. If none of the devices in the cluster can be connected an exception of type CryptoServerException will be thrown.

Parameters
devicesArray of CryptoServer device addresses. Each address may contain the prefix "TCP:" followed by the IP address or port number and IP address (seperated by a '@'), or the prefix "PCI:" followed by the device name of a PCI device.
Examples:
  • "TCP:192.168.4.99"
  • "TCP:288@192.168.4.201"
  • "PCI:0"
  • "PCI:/dev/cs2"
The array may consists of only one device.
connect_timeoutTimeout in milliseconds to establich the connection to a CryptoServer.
If the given value is less or equal to zero a default value of DEF_CONNECT_TIMEOUT is used.
read_timeoutRead timeout on command execution in milliseconds (see timeout).
If the given value is less or equal to zero a default value of DEF_READ_TIMEOUT is used.
Exceptions
CryptoServerExceptionif none of the devices in the cluster can be connected.

CryptoServerCluster ( CryptoServerConfig^  config)

Creates a new connection to a CryptoServer.

The configuration values will be read from a configuration object (see CryptoServerConfig).

Parameters
configConfiguration object. The following key names will be recognized:

Key Mandatory Default Value Description
Device Yes - Device specifier of CryptoServer to be connected.
Optionally multiple device specifier can be assigned in order to establish a fault-tolerant connection to a cluster of CryptoServer (failover).
The values have to be separated either by a space or tab character or have to be spread over multiple lines each ended by a backslash character (see CryptoServerConfig).
Timeout No 60000 Timeout in milliseconds the API waits for an answer on command execution (see timeout)
ConnectionTimeout No 3000 Timeout in milliseconds the API waits on attempt to connect to a CryptoServer
EndSessionOnShutdown No false Enables or disables an event handler that terminates active sessions on shutdown (see endSessionOnShutdown)
KeepSessionAlive No false Prevents a secure messaging session from expiring after having been idle for more than 15 minutes (see keepSessionAlive)
FallbackInterval No 0 Time in seconds the API waits before it tries to reconnect to the primary device (first device in the list) after having switched to another device due to a communication error (see fallbackInterval).
A value of '0' (default) specifies that the API should never switch back to the primary device.

Examplary configuration file:
# examplary configuration file for a CryptoServer cluster
Device = 192.168.4.183 \
192.168.4.185 \
192.168.4.186
ConnectionTimeout = 3000
#Timeout = 180000
Timeout = 30000
FallbackInterval = 600
KeepSessionAlive = true

void Logon ( String^  user,
String^  password 
)

Authenticates a user and establishs a secure messaging session with the current CryptoServer.

The API provides a failover mechanism that switches to another device in case of failure and returns as soon as a secure messaging session to a device could be established. Therefore user name and password will be stored internally in order to reestablish a session in case of a failure. If logon fails on all devices an exception of type CryptoServerException will be thrown.

Parameters
userName of the user who wants to autenticate to the CryptoServer.
passwordPassword of the user. May contain binary data.
Note
CryptoServerCluster only supports password authentication but no signature authentication.
Exceptions
CryptoServerExceptionif logon fails on all devices.
